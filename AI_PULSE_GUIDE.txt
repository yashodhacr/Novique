================================================================================
 NOVIQUE  —  The Real-Time AI Intelligence Platform
 Developer Guide  (updated: multi-source, real-time feed, topic filters)
================================================================================

WHAT IT IS
--------------------------------------------------------------------------------
Novique aggregates what's happening in AI from 10+ live sources, uses Claude
to summarize and score each item, and presents a fast, dark-mode feed. For
every item it answers:
    What happened?  Why does it matter?  Who is impacted?  What to watch next?

The full stack runs end-to-end across news, research, and open-source activity.
New sources plug into a single fetcher contract — nothing downstream changes.


FEATURES
--------------------------------------------------------------------------------
[x] Multi-source real-time ingestion (10 sources, polled every 60 seconds)

      NEWS  (every 60 seconds)
      - Hacker News          AI/LLM stories via Algolia API (points > 5)
      - Reddit               r/MachineLearning, r/artificial, r/LocalLLaMA, r/OpenAI
      - VentureBeat AI       RSS feed
      - TechCrunch AI        RSS feed
      - The Verge AI         RSS feed
      - Wired AI             RSS feed
      - AI News              RSS feed (artificialintelligence-news.com)
      - MIT Tech Review      RSS feed
      - IEEE Spectrum        RSS feed
      - Dev.to               AI-tagged articles via public REST API
      - GitHub Trending      AI/ML repos pushed in last 24h via GitHub search API

      RESEARCH  (every 30 minutes)
      - arXiv                cs.AI / cs.LG / cs.CL, enriched with Semantic Scholar
                             citation counts

      Links shared across multiple sources deduplicate to one entry automatically
      (SHA-256 of the normalized URL).

[x] AI Explainer Engine  (Claude claude-opus-4-8)
      For each item: 30-second summary, key takeaways, why it matters, who is
      impacted, what to watch next, topic tags, and sentiment. For arXiv papers,
      the contribution is explained in plain English.
      Without an API key the pipeline runs with a deterministic heuristic fallback.

[x] Proprietary scoring engine
      - AI IMPACT SCORE (0-100): significance. Blends source authority, engagement,
        entity importance (GPT/Claude/Anthropic/...), and content signals
        (launch/funding/breakthrough...). Recency-agnostic.
      - AI TREND SCORE (0-100): momentum. Blends engagement velocity (points/hour),
        recency boost, and topic co-occurrence across the batch. Decays hourly.

[x] Automatic front-end refresh  (every 60 seconds)
      React Query refetches the feed on a 60-second interval. Users see new
      articles without reloading the page. The hero countdown timer reflects
      the real 60-second ingest cycle.

[x] Topic / interest filter chips  (Signals page)
      AI-generated topic tags (from the Claude explainer) are surfaced as
      clickable chips below the kind selector. Multi-select — pick "LLMs" +
      "Funding" to see only articles tagged with either. "Clear all" resets.
      Filtering is instant, client-side.

[x] Time-aware greeting
      The hero greeting reads the visitor's browser clock and says
      "Good Morning / Good Afternoon / Good Evening / Good Night" correctly
      in any timezone without a server roundtrip.

[x] Feed sort + kind filters
      - Sort  : Top Impact / Trending / Latest
      - Kind  : All / News / Research Papers
      - Mode  : Discover (public) / For You (personalised, signed-in only)

[x] Personalized "For You" feed
      Signed-in users get the corpus re-ranked. Two signals:
        - Topics you FOLLOW (tap any topic tag)
        - Articles you SAVE  (tap the star; saved articles' topics count too)

[x] Accounts  (JWT auth, bcrypt passwords)
      Sign up / sign in from the header. Access + refresh tokens. A role field
      is in place for future admin/RBAC.

[x] Clean + dedupe
      URL normalization strips tracking params, www, and trailing slashes so
      the same story arriving from different sources collapses to one entry.

[x] REST API  (FastAPI, auto-generated OpenAPI docs at /docs)


TECH STACK
--------------------------------------------------------------------------------
Frontend   Next.js 15, TypeScript, Tailwind CSS, React Query
Backend    FastAPI (Python 3.12), SQLAlchemy 2, Pydantic v2

================================================================================
 OPTION A — Run with Docker Compose  (easiest, zero local installs)
================================================================================

PREREQUISITES
  - Docker Desktop running (the only hard requirement)
  - Optional: Anthropic API key for real AI summaries

STEPS
  1. Clone / open the project folder (the one with docker-compose.yml).

  2. (Optional) enable AI summaries:
         cp .env.example .env
     Open .env and paste your key:
         ANTHROPIC_API_KEY=sk-ant-...

  3. Build and start everything:
         docker compose up --build
     First build: a few minutes. Subsequent starts: a few seconds.

  4. Seed the database (two options):
       a) Wait — the Celery worker auto-ingests within ~60 seconds.
       b) Trigger one cycle right now:
              curl -X POST http://localhost:8000/api/ingest

  5. Open  http://localhost:3000

  To stop:           Ctrl+C  then  docker compose down
  To wipe the DB:    docker compose down -v

WHEN YOU CHANGE PYTHON DEPENDENCIES  (e.g. added feedparser)
  Python packages are baked into the Docker image at build time.
  Any time requirements.txt changes you must rebuild the backend image:
      docker compose build backend
      docker compose up
  This is a one-time rebuild per dependency change, not every run.

  Shortcut to install a single package without a full rebuild (temporary — lost
  on next container restart, useful for quick testing):
      docker compose exec backend pip install feedparser==6.0.11
      docker compose exec worker  pip install feedparser==6.0.11


================================================================================
 OPTION B — Run locally, no Docker  (good for active development)
================================================================================

PREREQUISITES
  Install these once on your machine:

  PostgreSQL 16
      Windows:  winget install PostgreSQL.PostgreSQL.16
                or download installer from https://www.postgresql.org/download/
      macOS:    brew install postgresql@16 && brew services start postgresql@16
      Linux:    sudo apt install postgresql-16

  Redis 7
      Windows:  Option 1 — WSL2: wsl --install, then inside WSL: sudo apt install redis
                Option 2 — Memurai (native Windows Redis): https://www.memurai.com/
                Option 3 — use Docker just for Redis:
                             docker run -d -p 6379:6379 redis:7-alpine
      macOS:    brew install redis && brew services start redis
      Linux:    sudo apt install redis-server && sudo systemctl start redis

  Python 3.12+
      Windows:  winget install Python.Python.3.12
      macOS/Linux: brew install python@3.12  /  sudo apt install python3.12

  Node.js 20+
      Windows/macOS/Linux: https://nodejs.org or  nvm install 20

CREATE THE DATABASE (first time only)
  psql -U postgres
  CREATE USER aipulse WITH PASSWORD 'aipulse';
  CREATE DATABASE aipulse OWNER aipulse;
  \q

SET ENVIRONMENT VARIABLES
  Create backend/.env  (or export in each terminal):

      DATABASE_URL=postgresql+psycopg2://aipulse:aipulse@localhost:5432/aipulse
      REDIS_URL=redis://localhost:6379/0
      ANTHROPIC_API_KEY=sk-ant-...   # optional; omit for heuristic fallback
      JWT_SECRET=change-me-in-production-please-use-32-chars

BACKEND  (terminal 1 — API server)
  cd backend
  python -m venv .venv
  .venv\Scripts\activate          # Windows
  # source .venv/bin/activate     # macOS / Linux
  pip install -r requirements.txt
  python -m app.bootstrap         # creates tables
  uvicorn app.main:app --reload --port 8000

CELERY WORKER  (terminal 2 — scheduled ingestion)
  cd backend
  .venv\Scripts\activate          # Windows
  # source .venv/bin/activate     # macOS / Linux
  celery -A app.celery_app worker --beat --loglevel=info

FRONTEND  (terminal 3)
  cd frontend
  npm install
  npm run dev                     # http://localhost:3000

FIRST CONTENT
  Either wait ~60 seconds for the worker to auto-ingest, or trigger manually:
      curl -X POST http://localhost:8000/api/ingest

ADDING A NEW PYTHON PACKAGE  (local mode — no Docker needed)
  Just pip install it in the activated venv:
      pip install feedparser==6.0.11
  Then add it to requirements.txt so collaborators pick it up on next install.
  No rebuild step needed when running locally.


================================================================================
 HOW TO SHARE THE SITE
================================================================================

Same machine:
    http://localhost:3000

Others on the same Wi-Fi:
    1. Find your local IP: ipconfig (Windows) / ipconfig getifaddr en0 (macOS)
    2. Set NEXT_PUBLIC_API_URL=http://<your-ip>:8000 in frontend/.env.local
    3. Restart the frontend
    4. Share  http://<your-ip>:3000

Quick public demo (no server):
    Use ngrok to tunnel both ports:
        ngrok http 3000   (second tunnel for 8000)
    Set NEXT_PUBLIC_API_URL to the public 8000 URL, restart frontend.

Production deployment:
    Server (e.g. AWS EC2 / DigitalOcean) + Nginx + HTTPS
    Managed Postgres (RDS / Supabase), managed Redis (Upstash / ElastiCache)
    Run uvicorn behind gunicorn, serve Next.js with `npm run build && npm start`

 API ENDPOINTS
================================================================================

GET  /health
        Liveness check.  {"status":"ok"}

GET  /api/feed?sort=impact|trend|recent&kind=news|paper&limit=40
        Ranked feed. kind is optional (omit for mixed; kind=paper = Research Hub).
        Returns ArticleOut[] with full AI explainer fields.

POST /api/ingest
        Trigger one full ingest cycle manually.
        Returns {"fetched":N,"cleaned":N,"created":N,"updated":N}

POST /api/auth/register            {email, password}   -> {access_token, refresh_token}
POST /api/auth/login               {email, password}   -> {access_token, refresh_token}
POST /api/auth/refresh             {refresh_token}     -> {access_token}
GET  /api/me                       Bearer token        -> user profile
GET  /api/me/interests             Bearer token        -> [{topic}]

================================================================================
 TROUBLESHOOTING
================================================================================

"Could not reach the API" on the site:
    Backend isn't up. Confirm http://localhost:8000/health returns {"status":"ok"}.
    Local mode: check uvicorn is running and DATABASE_URL is set correctly.
    Docker mode: docker compose ps — all services should show "healthy".

Feed is empty:
    You haven't ingested yet. Run:  curl -X POST http://localhost:8000/api/ingest
    Or wait 60 seconds for the Celery worker to run automatically.

feedparser ImportError / ModuleNotFoundError:
    Local mode: pip install feedparser==6.0.11  (inside activated venv)
    Docker mode: docker compose build backend && docker compose up

Summaries look generic / say "ANTHROPIC_API_KEY not set":
    Add ANTHROPIC_API_KEY to .env (Docker) or export it in your terminal (local).
    The app still works without it — summaries use the heuristic fallback.

Port already in use (3000, 8000, 5432, 6379):
    Docker: edit port mappings in docker-compose.yml.
    Local: stop the conflicting process or change the port in the run command.

Redis connection refused (local mode):
    Windows: start Memurai service, or run Redis in WSL2: sudo service redis start
    macOS: brew services start redis

arXiv returned nothing:
    arXiv rate-limits occasionally. The pipeline tolerates a single source
    failing — other sources still ingest. Retry in a few minutes.

RSS feed returned nothing for one source:
    Some feeds (MIT Tech Review, Wired) occasionally change URLs or add
    paywalls. The per-feed try/except swallows the failure; other feeds
    still run. Check the Celery worker logs for the specific error.


================================================================================
 ADDING A NEW SOURCE  (developer notes)
================================================================================

1. Add a fetch_* function to  backend/app/ingestion/crawler.py
   The only contract: return a list[RawItem].
   RawItem fields: title, url, source, author, published_at, points,
                   num_comments, content (optional), kind ("news"|"paper"),
                   citation_count (papers only).

2. Import it in  backend/app/tasks.py  and add it to the fetchers=[] list
   in ingest_news() (for news cadence) or ingest_papers() (30-min cadence).

3. That's it. The pipeline (clean, summarize, score, store) is unchanged.

Example — adding Product Hunt AI:
    def fetch_producthunt() -> list[RawItem]:
        ...  # call their GraphQL API
    # then in tasks.py:
    fetchers=[..., fetch_producthunt]
